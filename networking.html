How Networking Makes My Website Work

Domain Name System (DNS)

This is a system used in IP networks. Simply put, it refers to the use of easy-to-remember domain names, as IP addresses, which are a series of numbers, are difficult to memorize. It facilitates internet use by converting domain names into IP addresses that can be accessed on IP networks.

My domain seoyoung161.github.io is resolved to an IP address so that users can access the server without memorizing numbers. Through the nslookup command, I confirmed my domain points to GitHub's server IPs.

Resolved IP (IPv6): 2606:50c0:8000::153
Resolved IP (IPv4): 185.199.108.153

IP Addressing

IP addressing identifies network interfaces, provides a host's location within a network, and is used to establish routing. IPv4 uses a 32-bit address space, while IPv6 uses a 128-bit address space. This website is accessible via both IPv4 and IPv6 addresses.

This website uses both IPv4 and IPv6 addresses. When I ping my domain, it responds via IPv6 (2606:50c0:8000::153), which is a 128-bit address space, ensuring modern connectivity.

Protocols & Status Codes

The website uses HTTPS over TCP/IP. TCP/IP is the underlying protocol of the Internet, enabling various data communications. It allows us to use various Internet functions, such as visiting websites and sending emails. HTTPS ensures that the data exchanged between the browser and the server is encrypted and secure.

This site enforces HTTPS (Hypertext Transfer Protocol Secure) to encrypt data. Using Browser Developer Tools, I verified the following:

  • Status Code: 200 OK : It means the request was successful.
  • Protocol: HTTP/2 or HTTP/3 (over TLS)
  • Server Address: [2606:50c0:8000::153]:443

You can see my 200 OK status and response headers in the Network tab of your browser's inspect tool.
